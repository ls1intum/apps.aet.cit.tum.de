# apps.aet.cit.tum.de

Hugo + Tailwind CSS static site listing apps and tools built by the AET chair at TU Munich.
Deployed to GitHub Pages at https://apps.aet.cit.tum.de via GitHub Actions.

## Tech stack

- Hugo Extended (static site generator)
- Tailwind CSS v3 with PostCSS
- No JavaScript framework; no CDN dependencies

## Local development

```bash
npm install
npm run dev        # http://localhost:1313
```

Use `npm run dev` (not bare `hugo server`) for two reasons: it puts `node_modules/.bin` on PATH so PostCSS is found, and it passes `--baseURL http://localhost:1313/` so the site is served at the root instead of the GitHub Pages subpath.

## Build

```bash
npm run build      # output in public/
```

`hugo_stats.json` is generated by Hugo during build (required for Tailwind JIT purging). It is gitignored.

## Adding an app

1. Open `data/apps.yaml`
2. Find the relevant section (or add a new one)
3. Append an entry:

```yaml
- name: "My App"
  logo: "/logos/my-app.svg"          # optional; falls back to initials if absent
  description: "One or two sentences."
  links:
    tool: "https://my-app.aet.cit.tum.de"   # optional
    docs: "https://docs.my-app.aet.cit.tum.de"  # optional
    repo: "https://github.com/ls1intum/my-app"   # optional
```

4. Add the logo SVG to `static/logos/` (40x40 px recommended)
5. Run `npm run dev` to verify locally

Omitting a `links.*` key hides the corresponding button entirely.

## Design tokens

| Token | Value | Usage |
|---|---|---|
| `primary` | `#154a84` | TUM blue; header background, primary buttons |
| `border` | `#e2e8f0` | Card borders, outline buttons |
| `shadow-card` | blue-tinted subtle shadow | App card elevation |

Component classes defined in `assets/css/main.css`:
- `.app-card` - white card with border and shadow
- `.btn-primary` - filled TUM-blue button
- `.btn-outline` - bordered secondary button

## File structure

```
assets/css/main.css        Tailwind entry point + component classes
content/                   Markdown content (home, privacy)
data/apps.yaml             App listing data - EDIT THIS to add apps
layouts/                   Hugo templates
  _default/baseof.html     HTML shell + CSS pipeline
  index.html               Homepage
  privacy/single.html      Privacy page
  partials/
    header.html
    footer.html
    app-card.html          Single app tile
    icons/                 Inline SVG icons (no CDN)
static/
  CNAME                    Custom domain
  images/aet-logo.svg      Commit this separately
  logos/                   App logos
```

## Deployment

Hosted on GitHub Pages. The workflow at `.github/workflows/deploy.yml` builds and deploys on every push to `main`.

**One-time setup:** In repository Settings > Pages, set **Source: GitHub Actions**.

The `static/CNAME` file is automatically copied to `public/CNAME` by Hugo, so the custom domain persists across deploys.

## AET logo

Commit `static/images/aet-logo.svg` to show the AET logo in the header.
The header gracefully hides the `<img>` element if the file is absent.
